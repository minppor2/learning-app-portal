# PRD.md  
# 학습용 웹앱 통합 포털  
## App Launcher Portal for Middle School Learning Web Apps

---

## 1. 프로젝트 개요

학습용 웹앱 통합 포털은 교사가 제작하거나 수업에 활용하는 다양한 웹앱을 한곳에 모아 학생과 교사가 쉽게 접근할 수 있도록 하는 웹 포털이다.

본 포털은 단순한 링크 모음이 아니라, **왼쪽 목차 메뉴 + 오른쪽 가로 스크롤 카드형 앱 런처 + 팝업 미리보기 + 실행하기 버튼** 구조를 가진다.

전체 디자인은 업로드된 디자인 시스템을 기반으로 한다.  
핵심 스타일은 다음과 같다.

- 딥 오버진 계열의 메인 컬러
- 크림·라벤더 계열의 부드러운 배경
- 파스텔 메쉬 그라데이션
- 흰색 카드형 UI
- 둥근 pill 버튼
- 깔끔한 SaaS형 포털 레이아웃
- 중학생에게 친근하지만 교사가 보기에도 전문적인 화면

---

## 2. 제품 목표

### 2-1. 핵심 목표

- 여러 학습용 웹앱을 하나의 포털에서 통합 제공한다.
- 학생이 원하는 앱을 시각적으로 쉽게 찾을 수 있도록 한다.
- 왼쪽 목차 메뉴를 통해 카테고리별 앱 탐색을 제공한다.
- 오른쪽 영역에서 카드 이미지를 가로로 넘기며 앱을 탐색할 수 있도록 한다.
- 앱 실행 전 팝업 미리보기로 앱의 목적과 사용 방법을 확인할 수 있도록 한다.
- 교사는 수업에 사용할 앱을 빠르게 안내하고 관리할 수 있도록 한다.

### 2-2. 성공 기준

- 학생이 첫 화면에서 원하는 앱을 10초 안에 찾을 수 있다.
- 앱 카드를 클릭하면 바로 이동하지 않고 미리보기 팝업이 열린다.
- 팝업에서 앱 설명, 학습 목표, 사용 방법, 실행 버튼을 확인할 수 있다.
- 교사는 오늘의 수업앱을 쉽게 지정할 수 있다.
- 전체 화면은 깔끔하고 정돈된 느낌을 준다.
- 중학생 대상이지만 지나치게 유치하지 않다.

## 3. 대상 사용자

### 3-1. 학생

- 중학생
- 정보, 인공지능, 디지털 윤리, 자유학기, 동아리 수업 참여 학생
- 텍스트보다 이미지와 카드형 UI를 더 직관적으로 이해하는 사용자
- 복잡한 메뉴보다 큰 버튼과 명확한 실행 흐름을 선호하는 사용자

### 3-2. 교사

- 정보 교과 교사
- 자유학기, 동아리, 창의적 체험활동 운영 교사
- 여러 웹앱 링크를 한곳에 정리하고 싶은 교사
- 수업 차시별로 앱을 안내하고 싶은 교사
- 학생들에게 매번 개별 링크를 전달하는 번거로움을 줄이고 싶은 교사

### 4-1. 한 줄 콘셉트

**중학생을 위한 학습 웹앱 앱스토어형 포털**

```text
┌──────────────────────────────────────────────┐
│ 상단 헤더                                     │

├──────────────┬───────────────────────────────┤
│ 왼쪽 목차     │ 오른쪽 앱 카드 런처             │
│              │                               │
│ 전체앱        │ [웹앱 카드] [웹앱 카드] →       │
│ 오늘의 수업앱 │                               │
```

```text

→ 카드를 오른쪽으로 넘기며 탐색
→ 원하는 웹앱 카드 클릭
→ 팝업 미리보기 확인
→ 실행하기 버튼 클릭
→ 웹앱 실행
```

## 5. 디자인 시스템 적용 방향

### 5-1. 디자인 키워드

본 포털은 다음 디자인 키워드를 따른다.

- 깔끔함
- 부드러움
- 앱스토어형
- 학교 포털형
- 전문적이지만 친근한 느낌
- 카드 중심 UI
- 파스텔 메쉬 배경
- 딥 오버진 포인트 컬러

- 팝업 실행 버튼
- 교사용 주요 액션 버튼


사용 위치:

- 전체 배경

- 앱 카드 배경
- 팝업 배경
- 카테고리 섹션 배경

사용 위치:

사용 위치:

- 활동지 링크

- 외부 웹앱 URL

### 5-3. 타이포그래피

폰트는 시스템 기본 폰트 또는 Inter 계열을 사용한다.

#### 제목

- 크고 명확하게
- 약간 좁은 자간 적용
- 포털 제목, 섹션 제목, 앱 카드 제목에 사용

#### 본문

- 16px 기준
- 줄간격 1.5 이상
- 학생이 읽기 편한 간결한 문장 사용

#### 버튼

- 굵은 글씨
- 짧고 명확한 문구 사용

예시:

```text
실행하기
미리보기
교사용 설명
오늘 수업앱
```

---

### 5-4. 버튼 스타일

모든 주요 버튼은 pill 형태로 통일한다.

```css
border-radius: 90px;
padding: 14px 28px;
font-weight: 700;
```

#### Primary Button

사용 위치:

- 실행하기
- 오늘의 수업앱 시작
- 앱 등록
- 저장하기

스타일:

```css
background: #4a154b;
color: #ffffff;
border-radius: 90px;
```

#### Secondary Button

사용 위치:

- 교사용 설명
- 자세히 보기
- 닫기
- 뒤로가기

스타일:

```css
background: #f9f0ff;
color: #1d1d1d;
border-radius: 90px;
```

#### Outline Button

사용 위치:

- 필터 초기화
- 앱 목록 보기
- 외부 링크 확인

스타일:

```css

```

### 5-5. 카드 스타일

앱 카드는 흰색 카드 기반으로 구성한다.

```css
background: #ffffff;
border-radius: 16px;
border: 1px solid #e6e6e6;
padding: 0;
overflow: hidden;
```

### 카드 구성

- 상단: 대표 이미지
- 중단: 카테고리, 앱 이름, 설명
- 하단: 태그, 미리보기 버튼

카드 내부 여백은 20px 내외로 설정한다.

---

### 5-6. 파스텔 메쉬 그라데이션

포털의 상단 히어로 영역과 오른쪽 카드 배경 일부에는 파스텔 메쉬 그라데이션을 사용한다.

#### 사용 목적

- 단조로운 학교 링크 모음 느낌을 줄인다.
- 앱스토어형 포털의 시각적 완성도를 높인다.
- 학생이 처음 접속했을 때 흥미를 느끼게 한다.

#### 예시 CSS

```css
background:
  radial-gradient(circle at 10% 20%, #f4ede4 0, transparent 35%),
  radial-gradient(circle at 80% 10%, #f9f0ff 0, transparent 30%),
  radial-gradient(circle at 50% 80%, #efe2ff 0, transparent 35%),
  #ffffff;
```

## 6. 주요 기능

### 6-1. 왼쪽 목차 메뉴

왼쪽 목차는 포털 탐색의 중심이다.

#### 메뉴 구성

- 전체앱
- 오늘의 수업앱
- 코딩
- 인공지능
- 디지털 윤리
- 퀴즈·복습
- 프로젝트 활동
- 학급도구
- 즐겨찾기
- 교사용 관리

#### 요구사항

- 선택된 메뉴는 딥 오버진 배경으로 강조한다.
- 선택되지 않은 메뉴는 흰색 또는 크림색 배경을 사용한다.
- 메뉴는 세로형으로 배치한다.
- 학생용 화면에서는 관리 메뉴를 숨긴다.
- 교사용 화면에서는 관리 메뉴를 보여준다.

---

### 6-2. 오른쪽 가로 스크롤 앱 카드

오른쪽 영역에는 앱 카드가 가로로 배치된다.

#### 요구사항

- 앱 카드는 오른쪽으로 스크롤하며 볼 수 있다.
- 카드에는 대표 이미지가 크게 들어간다.
- 한 화면에 3~4개 정도의 카드가 보이도록 한다.
- 모바일에서는 1~2개 카드가 보이도록 한다.
- 트랙패드, 마우스, 터치 스와이프를 지원한다.

#### 카드에 들어가는 정보

- 대표 이미지
- 카테고리
- 앱 이름
- 대상 학년
- 한 줄 설명
- 태그
- 상태 배지
- 미리보기 버튼

#### 카드 예시

```text
┌──────────────────────┐
│    대표 이미지         │
│    함수 디버깅 화면    │
├──────────────────────┤
│ 코딩 · 중1             │
│ 함수 디버깅 실습       │
│ 잘못된 블록을 찾아요   │
│ #함수 #디버깅          │
│ [미리보기]             │
└──────────────────────┘
```

---

### 6-3. 앱 미리보기 팝업

앱 카드를 클릭하면 팝업이 열린다.

#### 팝업 목적

학생이 앱을 실행하기 전에 다음을 확인할 수 있도록 한다.

- 어떤 앱인지
- 무엇을 배우는지
- 어떻게 사용하는지
- 얼마나 걸리는지
- 바로 실행해도 되는지

#### 학생용 팝업 구성

- 앱 이름
- 카테고리
- 대표 이미지 또는 미리보기 화면
- 앱 설명
- 학습 목표
- 사용 방법
- 예상 소요 시간
- 실행하기 버튼
- 닫기 버튼

#### 교사용 팝업 추가 구성

- 성취기준
- 수업 차시
- 준비물
- 활동 순서
- 평가 활용 가능 여부
- 교사용 메모
- 활동지 링크
- 공개 여부

#### 팝업 예시

```text
┌────────────────────────────┐
│ 함수 디버깅 실습             │
│ 코딩 · 중1 · 20분            │
├────────────────────────────┤
│ [앱 미리보기 이미지]          │
├────────────────────────────┤
│ 함수 안의 잘못된 블록을 찾아  │
│ 점프 동작을 수정하는 앱입니다. │
│                              │
│ 학습 목표                    │
│ - 함수의 역할을 이해한다.     │
│ - 오류가 생긴 블록을 찾는다.  │
│                              │
│ [실행하기] [교사용 설명] [닫기]│
└────────────────────────────┘
```

### 6-4. 앱 실행 방식

팝업에서 실행하기 버튼을 클릭하면 웹앱이 실행된다.

#### 권장 실행 방식

초기 버전에서는 **새 탭 실행**을 기본으로 한다.

이유:

- 구현이 간단하다.
- 외부 웹앱 보안 정책 문제를 줄일 수 있다.
- 포털 화면을 유지할 수 있다.
- 학생이 앱을 닫고 다시 포털로 돌아오기 쉽다.

#### iframe 미리보기

팝업 내부에는 가능한 경우 iframe으로 웹앱 미리보기를 표시한다.

단, 일부 외부 웹앱은 보안 정책상 iframe 표시가 제한될 수 있으므로, 다음과 같이 대체한다.

```text
미리보기를 표시할 수 없는 앱입니다.
실행하기 버튼을 눌러 새 탭에서 열어 주세요.
```

### 6-5. 검색 기능

상단 검색창에서 앱을 검색할 수 있다.

#### 검색 대상

- 앱 이름
- 카테고리
- 태그
- 설명
- 대상 학년
- 수업 키워드

#### 검색 예시

- 함수
- 디버깅
- AI
- 사이버폭력
- 퀴즈
- 카드뉴스
- 발표
- 모둠
- 알고리즘

---

### 6-6. 오늘의 수업앱

교사가 오늘 사용할 앱을 상단 또는 왼쪽 메뉴 첫 번째에 고정한다.

#### 표시 방식

- 오늘의 수업앱 카드는 더 크게 표시한다.
- 카드에 `오늘 사용` 배지를 표시한다.
- 왼쪽 메뉴에서 `오늘의 수업앱`을 선택하면 오늘 사용할 앱만 보인다.

#### 학생 화면

```text
오늘 사용할 앱입니다.
선생님 안내에 따라 실행해 주세요.
```

#### 교사 화면

```text
오늘의 수업앱으로 지정됨
수업 차시: 함수와 오류 수정
예상 시간: 20분
```

## 7. 학생용 화면 요구사항

학생용 화면은 최대한 단순하게 구성한다.

### 포함 요소

- 포털 제목
- 검색창
- 왼쪽 목차
- 오른쪽 앱 카드
- 가로 스크롤
- 앱 미리보기 팝업
- 실행하기 버튼

### 제외 요소

- 앱 등록
- 앱 수정
- 앱 삭제
- 공개 설정
- 교사용 메모
- 관리자 설정

### UX 원칙

- 앱 이름보다 대표 이미지를 먼저 보이게 한다.
- 설명은 짧게 작성한다.
- 실행하기 버튼은 크게 배치한다.
- 앱을 클릭하면 바로 이동하지 않고 팝업을 먼저 보여준다.
- 학생이 실수로 잘못 누르더라도 쉽게 닫을 수 있어야 한다.

---

## 8. 교사용 화면 요구사항

교사용 화면은 학생용 화면 위에 관리 기능을 추가한다.

### 포함 요소

- 앱 등록

## 11. 화면별 상세 설계

### 11-1. 메인 화면

#### 목적

학생이 학습용 웹앱을 빠르게 찾고 실행하는 첫 화면이다.

#### 구성

- 상단 헤더
- 검색창
- 왼쪽 목차
- 오른쪽 가로 스크롤 앱 카드
- 오늘의 수업앱 강조 영역

#### 디자인

- 배경은 크림·라벤더 계열의 부드러운 톤

- 앱 카드는 흰색 카드로 구성

## 14. 접근성 요구사항

- 모든 버튼에는 명확한 텍스트를 제공한다.
- 이미지에는 대체 텍스트를 제공한다.
- 색상만으로 상태를 구분하지 않는다.
- 키보드로 메뉴와 팝업을 이동할 수 있어야 한다.
- 팝업은 ESC 또는 닫기 버튼으로 닫을 수 있어야 한다.
- 글자 크기는 14px 이하로 과도하게 작아지지 않도록 한다.
- 실행하기 버튼은 최소 44px 이상의 클릭 영역을 가진다.

- 팝업은 화면 폭에 맞춰 표시

## 18. 최종 방향 요약

이 포털은 중학생을 위한 **학습용 웹앱 앱스토어**처럼 작동해야 한다.

학생은 왼쪽 목차에서 주제를 고르고, 오른쪽에서 카드 이미지를 넘기며 앱을 탐색한다.  
앱 카드를 클릭하면 팝업으로 미리보기를 확인하고, 실행하기 버튼을 눌러 웹앱을 시작한다.

디자인은 딥 오버진, 크림·라벤더 배경, 파스텔 메쉬 그라데이션, 둥근 pill 버튼, 흰색 카드형 UI를 기반으로 한다.

초기 버전은 기능을 복잡하게 만들기보다, 학생이 “쉽게 찾고, 미리 보고, 바로 실행하는 경험”에 집중한다.
